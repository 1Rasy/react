# STOCK-FLOW-SCOPE-001

## 状态

已确认。

## 产品决策

库存流水建设采用分阶段方案：当前先按方案 C 实施，最终目标为方案 B。

### 第一阶段：方案 C

- 建立一套可扩展的统一库存流水结构。
- 本阶段先接入“人工库存调整审核通过”产生的库存变化。
- 人工库存调整审核通过时，每个商品生成一条库存流水。
- 当前未接入统一流水的开单、售后、ERP 入库等既有库存逻辑暂时保持现状，不因本阶段改造被顺带改变。

### 最终目标：方案 B

后续所有正式库存变化都必须进入同一套库存流水，包括：

- 期初库存导入；
- 经销商 ERP 入库及后续差额修正；
- 销售开单扣减；
- 可继续销售的售后收回；
- 订单修改产生的库存差额；
- 订单删除产生的库存恢复；
- 人工库存调整。

## 统一结构约束

- 流水结构不能只为人工库存调整设计，必须预留通用的库存变化来源。
- 每条流水至少能够记录：业务员工号、商品条码、库存变化类型、来源单号、库存变化原因、变动数量、调整前库存、调整后库存、实际发生时间、操作人和创建时间。
- 库存变化类型和来源单号必须能够区分并追溯不同业务来源。
- 一项商品的一次实际库存变化对应一条流水，不把多个商品合并为一条。
- 流水写入后不得直接覆盖或删除；发生错误时通过新增反向或差额流水纠正。
- 第一阶段产生的数据必须能够直接沿用到最终方案 B，不得在接入其他库存类型时推翻或重建现有流水表。

## 导出兼容要求

- 库存变动明细导出统一读取这套库存流水。
- 第一阶段导出内容只包含已经接入流水的人工库存调整。
- 后续每接入一种库存变化类型，导出功能自动能够按该类型查询和导出，不另建独立导出数据表。
