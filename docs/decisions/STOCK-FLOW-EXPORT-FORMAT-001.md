# STOCK-FLOW-EXPORT-FORMAT-001

## 状态

已确认。

## 产品决策

库存变动明细导出采用方案 A。

## 文件格式

- 导出格式采用 Excel `.xlsx`。
- 第一阶段只提供 `.xlsx`，不要求同时提供 CSV。
- 文件中的商品条码必须按文本格式写入，保留全部位数及前导零，避免 Excel 自动显示为科学计数法。

## 导出字段及顺序

| 列 | 字段 | 规则 |
|---|---|---|
| A | 工号 | 发生库存变化的业务员工号 |
| B | 条码 | 商品条码，按文本格式导出 |
| C | 规格口味 | 使用商品 `spec + flavor` 组合展示 |
| D | 库存变化原因 | 展示该条流水的业务原因 |
| E | 库存变动数量 | 使用带正负号的原始变动数量；增加为正数，减少为负数 |
| F | 库存变动时间 | 该条库存变化实际生效的时间 |
| G | 库存变化类型 | 使用中文业务名称展示，不直接导出数据库英文代码 |
| H | 来源单号 | 人工调整申请单号、销售订单号、ERP 来源单号等 |
| I | 变动前库存 | 该条流水生效前的库存 |
| J | 变动后库存 | 该条流水生效后的库存 |
| K | 操作人 | 触发或审核该次库存变化的人员 |

## 日期与时间规则

- 导出日期范围按库存流水的实际发生时间筛选。
- 结束日期包含所选当天的全部记录。
- 例如选择 `2026-07-01` 至 `2026-07-12`，应包含北京时间 `2026-07-12 23:59:59.999` 之前发生的库存流水。
- 页面查询、数据库筛选和导出显示统一按中国标准时间 `Asia/Shanghai` 解释。

## 类型显示规则

- 库存变化类型在页面和导出文件中使用中文名称。
- 第一阶段至少包含 `人工库存调整`。
- 最终方案 B 接入后，类型扩展为期初库存、ERP 入库、ERP 差额修正、销售出库、售后可售收回、订单修改差额、订单删除恢复和人工库存调整等。

## 分阶段实施

- 第一阶段虽然只导出人工库存调整流水，但导出列、查询条件和文件结构直接按最终方案 B 建设。
- 后续新增库存变化类型时沿用同一个导出入口和同一套列结构，不重新设计文件格式。
