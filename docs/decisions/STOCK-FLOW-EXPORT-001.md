# STOCK-FLOW-EXPORT-001

## 状态

已确认。

## 业务目标

管理人员可以导出指定范围内的库存变动明细，用于按业务员或库存变化类型核对、追溯和线下分析。

## 筛选规则

- 导出前必须能够选择开始日期和结束日期。
- 日期范围用于筛选库存流水的实际发生时间。
- 结束日期包含所选当天的全部记录。
- 可以选择全部业务员，也可以指定业务员后导出。
- 可以选择全部库存变化类型，也可以指定类型后导出。
- 业务员筛选、库存变化类型筛选和日期范围可以组合使用。
- 导出的是符合筛选条件的逐条库存变动明细，不是按商品或人员汇总后的结果。
- 页面查询、数据库筛选和导出显示统一按中国标准时间 `Asia/Shanghai` 解释。

## 导出格式

- 导出采用 Excel `.xlsx` 格式。
- 第一阶段不要求同时提供 CSV。
- 条码必须按文本格式写入 Excel，保留全部位数和前导零，避免显示为科学计数法。

## 导出列及顺序

| 列 | 字段 | 规则 |
|---|---|---|
| A | 工号 | 发生库存变化的业务员工号 |
| B | 条码 | 商品条码，按文本格式导出 |
| C | 规格口味 | 使用商品 `spec + flavor` 组合展示 |
| D | 库存变化原因 | 展示该条库存变化的业务原因 |
| E | 库存变动数量 | 使用带正负号的原始变动数量；增加为正数，减少为负数 |
| F | 库存变动时间 | 该条库存变化实际生效的时间 |
| G | 库存变化类型 | 使用中文业务名称展示，不直接导出数据库英文代码 |
| H | 来源单号 | 人工调整申请单号、销售订单号、ERP 来源单号等 |
| I | 变动前库存 | 该条流水生效前的库存 |
| J | 变动后库存 | 该条流水生效后的库存 |
| K | 操作人 | 触发或审核该次库存变化的人员 |

## 分阶段数据范围

- 第一阶段按方案 C 实施时，导出只包含已接入统一流水的人工库存调整记录。
- 最终达到方案 B 后，同一个导出入口覆盖期初库存、ERP 入库、ERP 差额修正、销售出库、售后收回、订单修改差额、订单删除恢复和人工库存调整。
- 页面应根据当前已经接入的流水类型提供筛选选项，未接入的类型不得生成虚假明细。
- 第一阶段就按最终方案 B 的列结构建设，后续增加库存类型时不重新设计导出文件。

## 数据准确性约束

- 导出数量必须与库存流水中的原始正负变动数量一致，不在导出时重新计算库存变化。
- 同一条流水只能导出一次，不得因关联商品表或人员表产生重复行。
- 条码必须按文本处理，保留全部位数和前导零。
- C 列没有口味或规格时，不得显示 `null`、`undefined` 等技术字符。
- 导出结果必须遵守所选日期、业务员和库存变化类型筛选条件。
- 库存变化类型使用中文名称展示。
